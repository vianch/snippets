# Internationalization (i18next)

- **All user-facing copy goes through `t(...)` from `react-i18next`.** No hardcoded strings in components, no `Copy` const objects, no in-component string literals shown to the user.
- Locale files live in `public/locales/{language}/{namespace}.json`. One namespace per page/feature (e.g. `header.json`, `sheet.json`, `workspace.json`, `ui.json`, `common.json`).
- Namespace names live in `lib/constants/i18n.ts` as `I18nNamespaces.*` — components pass these constants to `useTranslation(...)`, never raw strings.
- Server components use `getServerTranslations(language, namespace)` from `lib/i18n/server.ts`.
- Client components use `const { t } = useTranslation(I18nNamespaces.X)` from `react-i18next`.
- Run `pnpm run i18n:extract` after adding new keys (uses `i18next-cli`). Run `pnpm run i18n:types` to regenerate the typed resources file.
- Interpolation values are passed as the second arg: `t("meta", { fileName, rows, columns })`. Never concatenate in JSX.
- Default fallback language is `en`. Add a new locale by dropping `public/locales/{code}/*.json` and extending `SupportedLanguages` in `lib/constants/i18n.ts`.
